## 关键CAN ID：
• 0x615 - 本机NM ID（网络管理）
• 0x720/0x728 - UDS诊断（物理寻址）
• 0x7DF - UDS功能寻址

## 串口通信 (cmd/)
用于与仪表SOC（Linux板）通信：
• 协议格式: [magic:0x2E][type][len][payload...][crc8]
• 主要功能: UI控制（图标显示/闪烁、车速、档位、SOC等）

## 🚀 程序流程
// main.c
1. 关闭看门狗
2. 初始化系统时钟 (d_clock)
3. 初始化GPIO (d_gpio)
4. 初始化中断
5. 初始化各模块（CAN、UART、ADC、PWM等）
6. 进入主循环 while(1)
    ├── 执行主状态机 (main_state_loop)
    ├── 执行电源管理 (pm_loop)
    ├── 执行网络管理 (nw_loop)
    ├── 执行CAN应用 (p_can_app_loop)
    ├── 执行串口命令 (p_cmd_loop)
    └── 喂看门狗

main(调度) -> state(模式) -> can(输入) -> uart_matrix(映射) -> p_cmd(封包) -> d_uart(发包) -> SDK(硬件细节)

## 代码功能

### UART主要函数
驱动文件（初始化）在/drv/d_uart.c中
用结构体信息+初始化函数
g_uart_dev[UART_ID_NUM];
drv_debug_init();// 初始化串口(调试串口)
drv_uart_init();// 初始化DMA串口

普通轮询发送：
`drv_debug_send_char()` 发送单个字符
DMA+fifo发送：
`drv_uart_send()`写入fifo
`drv_uart_send_task()` DMA发送数据


然后对于串口屏来说，和驱动层之间还有一层**协议层**
发送命令到屏幕：cmd_val_set(...)
（uart_matrix.c）uart_pub_msg_data 把命令转为对应的协议数据，并发送
drv_uart_send() 存到fifo，等待dma发送
- drv_uart_send_task() 轮询发送fifo中的数据

### 数据
消息两层分装，**状态**和**数据**
typedef struct __can_loop_t {
    uint8_t id;
    uint8_t dev_id;  
    uint8_t status;     // 状态
    uint8_t res;
    timer_fl_t timer;
    can_msg_t can_msg;  // 具体的can数据,cm_put_data()来解码
} can_loop_t; 

```c
typedef struct __can_msg_t {
    uint32_t    id;                     // can id
    uint8_t     dev_no;                 // CAN index(索引号)： CAN 1/2
    uint8_t     id_type;                // 0：标准帧   1: 扩展帧
    uint8_t     frame_type;             // 0：数据帧   1: 远程帧
    uint8_t     dlc;                    // 帧长度
    uint8_t     data[8]; 
} can_msg_t;
```

```c
int _write(int fd, const char* buf, size_t nbyte)
{
#ifdef __DEBUG
    uint32_t u32Index;
    uint8_t u8RetVal;
    uart_dev_t* pdev = &g_uart_dev[UART_DEBUG];

    for (u32Index = 0U; u32Index < nbyte; u32Index++) {
        u8RetVal = drv_debug_send_char(pdev->uart_type, (unsigned char)buf[u32Index], 0xFFFFFFFFU);
        if (u8RetVal != 1U) {
            break;
        }
    }

    (void)fd;
#endif
    return 1U ? 1 : 0;
}
```
